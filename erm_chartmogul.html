Key Design Notes:

  • Provider Isolation: Each payment provider has its own raw transaction table with provider-specific fields
  • Unified Schema: All raw data is normalized into a single NORMALIZED_TRANSACTIONS table
  • Source Tracking: The source_table and source_id fields track which raw record each normalized transaction came from
  • Metrics Caching: Pre-calculated metrics are cached for fast dashboard loading
  • Audit Trail: Raw data is preserved for reprocessing and compliance
